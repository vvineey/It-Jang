# 잇장

> 공학설계기초 1조 발표용 프로젝트

잇장은 가지고 있는 옷을 다시 활용하고, 필요한 옷은 신중하게 구매하도록 돕는 지속가능 코디 추천 서비스입니다.

## 개발 오케스트레이터

잇장 개발은 기능 구현만으로 끝나지 않습니다. 요구사항 정리, UX 문구 작성, UI 설계, 구현, 검수, 리뷰, Git 관리, 문서화를 하나의 작업 흐름으로 관리합니다.

| 역할 | 주요 책임 |
|---|---|
| 기획 | 서비스 요구사항 정리, 기능 우선순위 정의 |
| UX Writing | 사용자에게 자연스러운 화면 문구 작성 |
| UI Design | 모바일 화면 구성, 색상 및 여백 일관성 검토 |
| Frontend | React 기반 화면 및 컴포넌트 구현 |
| QA | 기능 동작, 라우팅, 반응형 화면 검수 |
| Review | 코드 구조, 중복, 네이밍, 문서화 여부 검토 |
| Git 관리 | 브랜치 전략, 커밋 메시지, PR 본문 작성 |
| 문서화 | README, 기능 설명, 추후 연동 계획 정리 |

### 역할별 검토 기준

#### 기획

- 요청을 기능 단위로 정리하고 목적, 범위, 우선순위를 정의합니다.
- 기능이 `내 옷장 활용`, `지속가능 소비`, `신중한 구매`라는 서비스 정체성과 맞는지 확인합니다.
- 작업 전에 사용자 스토리와 완료 기준을 작성합니다.

#### UX Writing

- 문구를 짧고 자연스러운 한국어로 작성합니다.
- 기능명이 아닌 사용자 행동 중심의 버튼 문구를 사용합니다.
- 기획서나 AI 데모처럼 보이는 딱딱하고 긴 문장을 피합니다.

#### UI Design

- 딥그린, 세이지 그린, 크림 색상 팔레트를 유지합니다.
- 모바일 화면에서 정보 위계와 주요 CTA가 명확하도록 구성합니다.
- 화면 목적에 맞는 구성을 사용하고 같은 템플릿을 반복하지 않습니다.

#### Frontend

- 페이지와 재사용 UI 컴포넌트의 책임을 분리합니다.
- mock 데이터는 별도 파일 또는 명확한 상수로 관리합니다.
- 기존 프로젝트 구조, 네이밍, 라우팅 방식을 우선해서 사용합니다.

#### QA

- 버튼, 입력값, 탭, 라우팅을 시나리오별로 확인합니다.
- 빈 상태, 로딩 상태, 에러 상태를 확인합니다.
- 모바일 레이아웃, 내부 스크롤, 하단 탭바 겹침을 점검합니다.

#### Review

- 코드 중복, 컴포넌트 책임, 네이밍, 사용하지 않는 코드를 검토합니다.
- 요구사항과 구현 결과가 일치하는지 확인합니다.
- 서비스 정체성과 디자인 톤이 유지되었는지 확인합니다.

#### Git 관리

- 작업 목적에 맞는 브랜치를 생성합니다.
- 의미 단위로 커밋을 나누고 Conventional Commit 형식을 사용합니다.
- 변경사항, 테스트 결과, 체크리스트가 포함된 PR 본문을 작성합니다.

#### 문서화

- 새 기능, 실행 방법, mock 데이터, 추후 API 연동 계획을 README에 반영합니다.
- 현재 구현된 기능과 미구현 기능을 구분합니다.

## 서비스 원칙

- 새 상품보다 내 옷장 아이템을 먼저 보여줍니다.
- 안 입던 옷을 다시 활용할 수 있는 선택지를 제안합니다.
- 가격 비교에서는 판매가보다 배송비 포함 최종 예상가를 강조합니다.
- 얼굴 사진과 체형 정보는 사용 목적을 짧고 명확하게 안내합니다.
- AI 착용 이미지는 참고용이라는 점과 크레딧 사용 정보를 명확히 보여줍니다.

## 개발 워크플로

모든 작업은 아래 순서로 진행합니다.

1. **요구사항 파악**
   - 요청을 한 문장으로 요약합니다.
   - 목표와 우선순위를 정의합니다.
   - 포함 범위와 제외 범위를 구분합니다.
2. **브랜치 계획**
   - 기능, 수정, 리팩터링, 문서화 목적에 맞는 브랜치를 제안합니다.
3. **구현 계획**
   - 수정할 파일, 새 컴포넌트, 데이터 흐름, mock 데이터를 정리합니다.
4. **구현**
   - 기존 코드 스타일을 따르며 기능을 작은 단위로 구현합니다.
5. **자체 검수**
   - 빌드, 라우팅, 동작, 상태별 UI, 모바일 레이아웃, 문구를 확인합니다.
6. **개선**
   - 검수에서 발견한 문제와 UI 디테일을 보정합니다.
7. **결과 보고**
   - 변경 내용, 변경 파일, 테스트 결과, 남은 TODO를 정리합니다.
8. **Git/PR 문서 작성**
   - 커밋과 PR을 작성하고 체크리스트를 포함합니다.

## 실행 방법

### 요구 환경

- Node.js 20 이상
- npm

### 로컬 실행

```bash
cd frontend
npm install
npm run dev
```

```bash
cd frontend
npm run lint
npm run build
```

## Mock 데이터

백엔드 API가 준비되기 전까지 아래 데이터를 프런트엔드에서 사용합니다.

### `frontend/src/data/mockData.ts`

- 옷 카테고리와 내 옷장 아이템
- 추천 태그와 카테고리별 추천 후보
- 사용자 정보 선택지와 스타일 옵션
- 코디 커뮤니티 게시글
- 순환 옷장 거래 게시글

### `frontend/src/data/priceCompareMock.ts`

- 가격 비교 대상 상품
- 플랫폼별 판매가, 배송비, 최종 예상가
- 가격 이력
- 내 옷장과 어울리는 아이템
- 중고 및 유사 상품 대안

### `frontend/src/data/outfitCopyMock.ts`

- 공개 코디 아이템과 내 옷장 대체 결과
- 원본, 내 옷장, 유사 상품, 대체 불가 상태
- 내 옷장 재현도와 점수 기준
- 작성자 개인정보를 사용하지 않는 따라입기 정책

### `frontend/src/data/circularServicesMock.ts`

- 옷장 아이템별 `PRIVATE`, `FRIENDS`, `PUBLIC` 공개 범위
- 매너온도 신뢰 지표와 후기
- 의류 수거 대상과 상태
- 플리마켓 행사 및 참여 유형
- 리워드 적립 정책, 내역, 월간 순환 기록

AI 착용 미리보기의 프롬프트, 코디, 크레딧 정보는 현재 페이지 상수와 프런트엔드 상태로 동작합니다.

## 추후 API 연동 계획

| 영역 | 연동 계획 |
|---|---|
| 인증 | SNS OAuth 및 이메일 인증 API 연동 |
| 사용자 정보 | 프로필과 체형 정보 저장 및 수정 API 연동 |
| 내 옷장 | 옷 등록, 조회, 수정, 삭제 및 착용 기록 API 연동 |
| 구매내역 | 무신사, 29CM, 네이버쇼핑 구매내역 연동 검토 |
| 코디 추천 | 사용자 조건과 내 옷장을 반영하는 추천 API 연동 |
| 가격 비교 | 상품 검색, 플랫폼 가격, 배송비, 가격 이력 API 연동 |
| AI 착용 | 얼굴 및 체형 기반 이미지 생성, 크레딧 결제 API 연동 |
| 커뮤니티 | 코디 게시글, 좋아요, 저장, 업로드 API 연동 |
| 코디 따라입기 | 공개 범위 검증, 공개 의류 참조, 요청 사용자 시착 세션 API 연동 |
| 옷장 공개 | 의류 아이템별 공개 범위 저장 및 공개 참조 API 연동 |
| 매너온도 | 거래 완료, 응답, 약속, 후기, 신고 기반 신뢰도 API 연동 |
| 의류 수거 | 수거 가능 여부 검수, 방문 예약, 리워드 지급 API 연동 |
| 플리마켓 | 행사 목록, 참가 신청, 공개 의류 출품 API 연동 |
| 리워드 | 순환 활동별 적립·사용·소멸 정책 API 연동 |
| 순환 옷장 | 상품 등록, 거래 상태, 검색 및 채팅 API 연동 |

> 실제 API 연동 시 컴포넌트에 직접 요청 로직을 추가하기보다 service 계층을 만들고 현재 mock 데이터를 API 응답으로 교체합니다.
